I. Daftar Perintah Git
    1. Persiapan (Setup)
        git init: Mengaktifkan Git di folder proyek.

        git clone <url>: Mengunduh proyek dari server (GitHub).

        git config --global user.name "Nama": Mengatur identitas kamu.

        git config --global user.email "email@anda.com": Mengatur email kamu.

    2. Alur Kerja Harian (Make a Change)
        git status: Cek perubahan file.

        git add <file>: Memilih file untuk disimpan (staging).

        git add .: Memilih semua file sekaligus.

        git commit -m "pesan": Menyimpan perubahan ke dalam catatan sejarah.

        git commit --amend: Memperbaiki pesan commit terakhir.

    3. Melihat Sejarah (Observe)
        git log: Melihat riwayat commit.

        git log --oneline: Riwayat ringkas.

        git log --oneline --graph --all: Melihat sejarah dalam bentuk diagram grafis (Sangat berguna untuk melihat posisi branch).

        git diff: Melihat perbedaan baris kode.

        git show <id>: Detail perubahan pada commit tertentu.

        git blame <file>: Mengetahui siapa yang mengubah baris kode tertentu.

    4. Percabangan (Branching)
        git branch: Daftar cabang.

        git checkout -b <nama>: Membuat dan pindah ke cabang baru.

        git checkout <nama>: Pindah antar cabang.

        git merge <nama>: Menggabungkan cabang lain ke cabang aktif.

        git branch -d <nama>: Menghapus branch yang sudah di-merge.

    5. Sinkronisasi (Sync)

        git remote -v: Cek daftar server remote.

        git remote add origin <url>: Menghubungkan lokal dengan GitHub.

        git push -u origin <nama-branch>: Upload kode ke GitHub.

        git pull: Download & gabungkan kode dari GitHub ke lokal.

        git fetch: Cek perubahan dari GitHub (tanpa menggabungkan).

    6. Penyelamatan (Recovery & Stash)
        git stash: Simpan pekerjaan yang belum selesai sementara waktu.

        git stash pop: Ambil kembali pekerjaan yang tadi di-stash.

        git checkout -- <file>: Membatalkan perubahan pada file yang belum di-add.

        git reset --hard: Membatalkan semua perubahan yang belum di-commit.

        git revert <id>: Membatalkan perubahan dari commit tertentu (tetap aman).

II. Contoh Skenario (A-Z)
    Simulasikan contoh proyek "Aplikasi Belanja":
        Langkah 1: Inisialisasi

        Bash
            mkdir aplikasi-belanja
            cd aplikasi-belanja
            git init

        Langkah 2: Menulis Kode
            Kamu membuat file index.html. Cek statusnya:

        Bash
            git status
            # Git akan bilang: "index.html tidak terlacak"
        
        Langkah 3: Simpan (Commit)

        Bash
            git add index.html
            git commit -m "feat: tambah halaman index"
        
        Langkah 4: Cabang untuk Fitur Baru
            Tiba-tiba ingin buat fitur login tanpa merusak index.html:

        Bash
            git checkout -b fitur-login
            # Sekarang kamu berada di cabang 'fitur-login'
            # Buat file login.html
            git add .
            git commit -m "feat: tambah halaman login"
        
        Langkah 5: Gabungkan
            Setelah selesai dan berhasil, gabungkan ke cabang utama (main):

        Bash
            git checkout main
            git merge fitur-login
        
        Langkah 6: Kirim ke GitHub

        Bash
            git remote add origin https://github.com/akun-kamu/aplikasi-belanja.git
            git push -u origin main
